Self-Healing Node Network

How It Works — never go offline, never lose a request

01

Multi-Node Routing

Requests are routed to the best available node based on speed, reputation, and model availability. If one node goes down, traffic shifts instantly.

02

Reputation Scoring

Every node has an on-chain reputation score based on uptime, speed, and response quality. High-reputation nodes get more traffic and rewards.

03

Quality Verification

Responses are verified against quality benchmarks. Nodes serving bad responses get penalized. Users always get reliable output.

04

Health Dashboard

Real-time monitoring of all nodes, models, and network health. Transparent metrics anyone can verify.

Use the CLI

Install the CLI and start asking questions immediately. No configuration needed.

Python SDK

Use the Python SDK as a drop-in replacement for OpenAI. Same interface, free inference.

SDK pip install gitblock-sdk

How to Use GitBlock

Step-by-step guides for every use case

1

Chat Completion

Send a message and get an AI response. Works exactly like OpenAI's chat completions API.

from gitblock import GitBlock client = GitBlock() response = client.chat( model="llama-3.3-70b", messages=[ {"role": "system", "content": "You are helpful."}, {"role": "user", "content": "What is Python?"} ], temperature=0.7, max_tokens=500 ) print(response.content)
2

Streaming Responses

Get responses as they're generated. Perfect for real-time chat interfaces.

from gitblock import GitBlock client = GitBlock() stream = client.chat( model="mistral-7b", messages=[ {"role": "user", "content": "Write a poem"} ], stream=True ) for chunk in stream: print(chunk.delta, end="", flush=True)
3

Code Generation

Generate code in any language. Use coding-optimized models for best results.

from gitblock import GitBlock client = GitBlock() response = client.chat( model="deepseek-coder-33b", messages=[{ "role": "user", "content": "Write a Python function to sort a list" }] ) print(response.content)
4

Run a Node

Earn tokens by serving inference on your hardware. Any GPU works.

# Install node software pip install gitblock-node # Register your node gitblock-node register --wallet 0xYourAddr # Start serving gitblock-node start --model llama-3.3-70b # Check status gitblock-node status

Frequently Asked Questions

Everything you need to know about GitBlock

Is GitBlock really free?

Yes, 100% free for basic usage. No credit card, no subscription, no hidden fees. Node operators earn tokens for serving inference, but users pay nothing.

How is it free if AI costs money to run?

Node operators volunteer their GPU hardware and earn tokens as rewards. The network incentivizes compute contribution through token economics, making inference free for users.

What AI models are available?

All major open-source models: Llama 3.3, Mistral 7B, Gemma 4, DeepSeek Coder, Qwen 2.5, Phi-3, Yi 1.5, CodeLlama, and 80+ more. All free, all community-powered.

Is my data safe?

Yes. All prompts are end-to-end encrypted. They're decrypted only on the serving node, processed, and the response is encrypted back. No central server ever sees your data.

Can I use it as a drop-in replacement for OpenAI?

Yes. The API is OpenAI-compatible. Just change the base URL and API key. Same request format, same response format. One line of code to switch.

How do I run a node and earn tokens?

Install gitblock-node, register your hardware, and start serving models. Any GPU works — from gaming PCs to data center rigs. More powerful hardware = more requests = more rewards.

Can I deploy my own GitBlock network?

Absolutely. Everything is open source under MIT license. Fork the repo, configure your network, deploy your own nodes. Full documentation available on GitHub.
